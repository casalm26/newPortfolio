# Portfolio Website - Product Requirements Document (PRD)
*Senior PM Refined Version*

## Executive Summary

### Strategic Vision
Create a differentiated pixel-art portfolio that positions Caspian as an innovative generalist developer who thinks outside conventional boundaries. The site serves as both a professional showcase and a memorable brand experience that stands out in a crowded market of traditional portfolio sites.

### Value Proposition
"A portfolio that's as innovative as the developer behind it" - demonstrating technical skills, creative thinking, and attention to user experience through an interactive, game-inspired interface that recruiters and peers will remember.

### Business Objectives
1. **Primary**: Increase quality job opportunities and career advancement
2. **Secondary**: Build professional network and industry recognition
3. **Tertiary**: Establish thought leadership as an innovative generalist

## User Personas & Journey Mapping

### Primary Persona: Technical Recruiters (70% of traffic)
- **Context**: Scanning 50+ portfolios per day
- **Time Budget**: 15-30 seconds initial scan
- **Key Needs**: Quick skill assessment, contact info, standout factor
- **Success Metric**: Moves to shortlist (5% of viewed profiles)
- **Journey**: Landing → Skills scan → Projects glimpse → Contact/LinkedIn

### Secondary Persona: Hiring Managers (25% of traffic)
- **Context**: Evaluating 3-5 pre-screened candidates
- **Time Budget**: 2-3 minutes deep dive
- **Key Needs**: Project depth, problem-solving approach, cultural fit
- **Success Metric**: Invites to interview (20% of reviewed profiles)
- **Journey**: Direct to projects → CV timeline → About → Contact

### Tertiary Persona: Peers/Colleagues (5% of traffic)
- **Context**: Professional networking, referrals
- **Time Budget**: 1-5 minutes exploration
- **Key Needs**: Professional connection, shared interests
- **Success Metric**: Social engagement and referrals
- **Journey**: Variable → Social links → Projects → Game (retention)

## Visual Design System

### Pixel Art Specifications
- **Grid System**: 8px base grid for all elements, 16px for major layout components
- **Pixel Alignment**: All elements must align to grid for crisp rendering
- **Inspiration**: adventofcode.com aesthetic with game-like interface elements
- **Typography**: Monospace/pixel fonts (Courier New, Monaco, Menlo) with font-smooth: never
- **Borders**: Hard edges, no border-radius >2px, prefer 0px for authentic pixel look
- **Shadows**: Hard shadows with solid colors (2px 2px 0px) instead of blur effects

### Color System
**Dark Mode (Default) - Orange Theme:**
- Primary: #f97316 (main orange), #ea580c (hover), #c2410c (active)
- Background: #0f0f0f (true black for pixel aesthetic)
- Text: #ffffff (white), #d1d5db (muted)
- Accent borders and highlights in orange

**Light Mode - Purple/Blue Theme (Accessibility Optimized):**
- Primary: #6366f1 (purple-blue), #4f46e5 (hover), #4338ca (active)
- Background: #fafafa (soft white, not harsh bright)
- Text: #1f2937 (dark gray), #6b7280 (muted)
- Sufficient contrast ratios (4.5:1 minimum for normal text)

### Animation Guidelines
- **Transition Duration**: 50-150ms for snappy, game-like feel
- **Easing**: ease-out for responsive interactions
- **Progressive Loading**: Typewriter effects for text, block-by-block for images
- **Microinteractions**: 3D button press effects, color shifts on hover

## Technical Architecture

### Core Stack
- **Framework**: Next.js 15 (App Router)
- **Styling**: Tailwind CSS + Custom pixel-art components
- **Animations**: Framer Motion + CSS animations
- **Content Management**: Contentlayer2 for projects/blog
- **Deployment**: Vercel (recommended) or Netlify

### Data Architecture
```
/data/
├── projects/           # MDX files for projects
├── cv-timeline/       # JSON for career timeline
├── config/            # Site configuration
└── assets/            # Pixel art and media
```

### Performance Budget

- **FID**: < 100ms
- **CLS**: < 0.1
- **Bundle Size**: < 500KB initial load
- **Image Optimization**: WebP with fallbacks

### Mini Game Design Document (Pixel Runner)
**Core Concept**: Chrome dinosaur game reimagined with pixel-art character

**Visual Design**:
- **Character**: 16x16px pixel-art figure (running animation: 3-4 frames)
- **Environment**: Minimalist pixel landscape with parallax scrolling background
- **Obstacles**: Various pixel platforms/blocks moving left to right at different speeds
- **UI Elements**: Retro-style score counter, high score display, game over overlay

**Gameplay Mechanics**:
- **Movement**: Character runs continuously, player controls jumping only
- **Jumping**: Arc-based physics with gravity (not floaty)
- **Obstacle Generation**: Randomized platform heights and gaps
- **Difficulty Scaling**: Gradually increasing speed and obstacle frequency
- **Collision Detection**: Pixel-perfect collision with obstacles

**Controls & Accessibility**:
- **Desktop**: Spacebar or Up arrow to jump
- **Mobile**: Touch anywhere on screen to jump
- **Game State**: Disable page scroll when game is active
- **Accessibility**: Visual indicators for actions, keyboard-only playable

**Technical Requirements**:
- **Performance**: 60fps on mobile devices
- **Responsive**: Scales to different screen sizes without breaking pixel alignment
- **State Management**: Game state, score tracking, high score persistence
- **Integration**: Accessible from navigation, potential easter egg triggers

**UI Components**:
- **Start Screen**: Simple "Press Space to Start" with character preview
- **Score Display**: Real-time score in pixel font
- **Game Over Screen**: Final score, high score, restart option
- **High Score**: Local storage for persistent high scores
